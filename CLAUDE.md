# CLAUDE.md — Биржа рассылок (site_contacts)

## Быстрый старт

```bash
python manage.py runserver          # Dev-сервер
python manage.py migrate            # Миграции
python manage.py collectstatic --noinput  # Статика
```

## Стек

- **Python 3.10+**, **Django 4.2+**, **PostgreSQL** (psycopg2-binary)
- **S3** (Timeweb Cloud) для медиа-файлов через django-storages + boto3
- **WhiteNoise** для статики в production, **Gunicorn** как WSGI-сервер
- **openpyxl** — импорт/экспорт Excel, **Pillow** — изображения, **imageio-ffmpeg** — сжатие видео

## Суть проекта

Биржа рассылок. Основной цикл работы:

1. **Админы** загружают контактные базы на сайт
2. **Пользователи** скачивают базу и делают рассылки по контактам
3. **Пользователи** отправляют отчёт (лид) о проделанной работе
4. **Админы/саппорт** проверяют отчёты (approve/reject/rework)
5. **Пользователи** получают оплату за одобренные отчёты, выводят баланс

### Отдельные ветки сайта

- **СС-админы (standalone_admin)** — полностью отдельная ветка. У них своя задача, своя команда воркеров, свои задания и отчёты. Не пересекаются с основной биржей.
- **Партнёрские админы (partner)** — работают сами по себе, по сути миррор основного сайта. Инвайтят пользователей по реферальной ссылке. Приглашённые работают как обычные пользователи (проверяют их обычные админы), а партнёрский админ получает процент с каждого одобренного отчёта своих рефералов.

## Роли пользователей

| Роль | Описание |
|------|----------|
| `user` | Пользователь — скачивает базы, делает рассылки, отправляет отчёты (лиды), выводит баланс |
| `support` | Поддержка — модерация отчётов, ответы на тикеты |
| `admin` | Администратор — полный доступ, управление пользователями, загрузка баз, модерация |
| `balance_admin` | Финансовый админ — обработка выплат, управление балансами |
| `standalone_admin` | СС-админ — отдельная ветка: своя команда воркеров, свои задания и отчёты |
| `worker` | Воркер (СС-ветка) — выполняет задания от standalone_admin, шлёт самостоятельные лиды |
| `partner` | Партнёрский админ — инвайтит пользователей, получает процент с их одобренных отчётов |

## Ключевые модели

- **User** — кастомная модель (AbstractUser) с role, status, balance, telegram_id, partner_owner, standalone_admin_owner
- **Lead** — отчёт пользователя о рассылке (статус: pending/approved/rejected/rework), normalized_contact для дубликатов
- **Contact / BaseType** — контактные базы для рассылок с лимитами выдачи в день
- **WorkerSelfLead** — самостоятельный лид воркера (ветка СС-админа, отдельная таблица от Lead)
- **LeadAssignment / WorkerReport** — задания и отчёты воркеров (ветка СС-админа)
- **SupportThread / SupportMessage** — система поддержки
- **WithdrawalRequest / WorkerWithdrawalRequest** — заявки на вывод

## Проверка дубликатов

Функция `_lead_exists_globally()` в `views.py` — нормализует контакт и проверяет кросс-платформенно (telegram/vk/ig/ok). Аналогичная `_self_lead_duplicate_exists()` в `views_worker.py` для воркеров СС-ветки.

## Маршруты (основные группы)

- `/` — Лендинг
- `/dashboard/` — Дашборд (контент зависит от роли)
- `/leads/report/` — Отправка отчёта о рассылке
- `/staff/*` — Админ-панель (модерация, пользователи, базы, статистика)
- `/staff/standalone/*` — Панель СС-админа (воркеры, задания, отчёты)
- `/worker/*` — Кабинет воркера (ветка СС-админа)
- `/partner/*` — Кабинет партнёрского админа
- `/support/*` — Поддержка
- `/admin/` — Django admin (is_staff=True)
